# 计算机视觉基础知识学习指南

本仓库面向实验室成员的计算机视觉基础学习，按子领域组织阅读路线、经典论文、代码仓库和实践建议。建议先建立图像分类与深度学习 backbone 的基础，再进入检测、分割、增强、VPR 和线框解析等方向。

## 1. 学习地图
| 子领域 | 入口文件 | 核心问题 | 先修基础 | 推荐定位 |
| --- | --- | --- | --- | --- |

| 目标检测 | [目标检测/readme.md](目标检测/readme.md) | 图像里有什么，在哪里 | 图像分类、IoU、NMS | 进入定位与实例级理解 |

## 2. 建议学习顺序
### 2.1 通用基础路线
1. [图像分类](图像分类/readme.md)：先掌握 CNN、ResNet、ViT 和预训练 backbone。
2. [目标检测](目标检测/readme.md)：学习从分类到定位的扩展，掌握框预测、NMS 和 mAP。
3. [语义分割](语义分割/readme.md)：学习像素级密集预测、多尺度上下文和 mask 表示。
4. [图像增强](图像增强/readme.md)：补充低层视觉能力，理解图像退化、复原和质量评价。

### 2.2 定位与几何路线
1. [图像分类](图像分类/readme.md)：建立 backbone 和表征学习基础。
2. [VPR](VPR/readme.md)：学习图像检索、地点识别和全局描述子。
3. [线框解析](线框解析/readme.md)：学习线段、junction 和结构化几何表示。
4. 将 VPR 与线框/局部特征结合，进一步进入视觉定位、SLAM、三维重建。

### 2.3 工程实践路线
1. 用 `torchvision` / `timm` 跑通分类训练和预训练权重加载。
2. 用 `mmdetection` 或 `detectron2` 跑通检测数据集训练。
3. 用 `mmsegmentation` 跑通语义分割训练和可视化。
4. 用 `BasicSR` / `SwinIR` / `Restormer` 跑通图像增强推理。
5. 用 `CosPlace` / `MixVPR` / `HLoc` 跑通地点检索与定位 demo。

## 4. 课程讲义使用方式
- 每个子目录都按“学习目标 → 发展脉络 → 方法地图 → 入门路线 → 进阶路线 → 论文精读 → 代码实践”组织。
- 入门阶段优先理解任务定义、经典模型和评价指标，不必一次读完所有细节。
- 进阶阶段重点比较方法之间的动机差异、结构差异和适用场景。
- 论文精读建议遵循“问题背景 → 方法结构 → 损失函数 → 实验设置 → 局限性”的顺序。
- 代码实践建议先跑官方 demo，再改数据集和配置，最后再尝试复现实验指标。

## 6. 论文精读模板
建议每篇论文都按以下问题整理笔记：

1. 这篇论文解决了什么问题？之前方法的主要缺陷是什么？
2. 论文的核心 idea 是什么？能否用一句话概括？
3. 方法结构包括哪些模块？输入输出分别是什么？
4. 损失函数、训练数据和评价指标是什么？
5. 实验中最关键的 ablation 是什么？证明了什么？
6. 这篇论文适合复现吗？有哪些代码仓库可以参考？
7. 它对当前课题有什么启发？可以迁移到哪里？

## 7. 代码复现模板
建议每个代码仓库都按以下顺序上手：

1. 阅读 `README`，确认环境、数据集、预训练权重和许可证。
2. 跑通官方 demo 或 inference 脚本，保存输入输出样例。
3. 跑通最小训练配置，记录显存、速度、日志和指标。
4. 修改数据路径和类别配置，迁移到自己的数据集。
5. 固定随机种子，记录配置文件、commit id 和实验结果。
6. 做 1—2 个小 ablation，确认关键模块是否真的有效。

## 8. 维护建议
- 新增论文时优先加入对应子目录的“论文精读”表格，而不是只堆链接。
- 新增代码仓库时说明它覆盖的任务、推荐用途和是否为官方实现。
- 每个方向都建议保留“入门”和“进阶”两条路线，方便不同基础的同学使用。
- 如果实验室课题发生变化，可以在对应目录继续增加“专题阅读”小节。
